✓ Core Components

  1. build_gt_distribution.py
     - Builds ground-truth temporal distribution for backchannel task
     - Reads manifest, extracts all backchannel events, bins to 0.2s windows
     - Output matches original gt_distribution.json exactly (verified)
     - Input: manifest JSON (backchannel only)
     - Output: gt_distribution.json with normalized sample IDs

  2. materialize_fdb.py
     - Creates FDB directory structure for all 4 tasks
     - Generates metadata.json per sample (task-specific fields)
     - Symlinks input.wav from source wavefiles
     - For backchannel: includes gt_distribution.json at root
     - Input: manifest JSON (any task), optional GT distribution
     - Output: FDB directories ready for model outputs + evaluation

  3. csv_to_manifest.py
     - Converts scored CSVs to lightweight manifest format
     - Filters by req_ok=True and quality_score threshold
     - Extracts task-specific timing and event information
     - Input: scored CSV files
     - Output: manifest JSON files

  - GT Distribution: Uses relative times from clip_origin, 0.2s bins, epsilon smoothing (1e-10)
  - Manifest Format: JSON with task-specific field schemas
  - Sample Filtering: req_ok=True + quality_score threshold in manifests
  - Audio Linking: Symlinks (not copies) to preserve disk space
